## Known Issues to Preserve

- PMDB route must keep MDB/ADB/TWT behavior unchanged.
- Arbitrum routes are Fintheon Arbitrum, not Ethereum Arbitrum.
- No auto-trading. Trades are proposals only; humans approve/deny.
- Supabase RLS cannot be weakened to fix calendar/proposal writes.

## Implementation Steps

1. Reproduce `.ics` ingest by POSTing a small valid `.ics` payload to `/api/desk/calendar/ingest-ics` and reading `/api/desk/calendar/queue`.
2. Verify `desk_calendar_events` migration and RLS policy. If auth blocks the intended user path, add a new migration with narrow user-scoped policy.
3. Ensure ingest returns a UI-useful shape: `{ ingested, events, statusMessage }` or equivalent.
4. Add diagnostic fields for desk calendar last ingest, queue count, and latest error.
5. Add backend route or econ route mode for countdown test events if frontend needs a real backend event instead of local mock.
6. Verify Arbitrum seat construction includes persona prompt, role, visible display name, role subtitle, weight, temperature, and model.
7. Update backend seat labels to Harper/Oracle/Feucht/Consul/Herald and role subtitles. Keep serialized role compatibility if existing DB rows use older labels.
8. Replace `Neutral` with `Chop` in Arbitrum-facing backend summaries where generated by backend code/prompts.
9. Trace PMDB path in `brief-generator.ts`. Ensure latest session-trigger verdict within the intended day/session window is injected into PMDB, not stale prior verdict.
10. Add a diagnostic field comparing latest PMDB timestamp and latest Arbitrum verdict timestamp.
11. Add or verify proposal resolution job/route: resolved prediction-market proposals write win/loss, resolver timestamp, market close countdown, and agent attribution.
12. Add Econ synthesis cache table if missing: event family, date range, selected event ids, raw normalized rows, synthesis text, model/version, created_at, updated_at, user_id nullable where needed.
13. Add/update changelog entry.
