## 🧾 Signal logging standard

When logging a signal to Signals:

- **Signal:** Descriptive title
- **Source:** Where the data came from
- **Type:** Fundamental
- **Strength:** Weak, Moderate, or Strong
- **Notes:** Impact analysis and which desks should pay attention

## 📅 When triggered from Economic Events

If triggered by a new page in the Economic Events Tracker ([](https://www.notion.so/ee319e74caf648f6843ba3019a8de97d?pvs=21)):

- Read the event details (Name, Importance, Tier, Category, Forecast, Previous)
- Assess the impact level using the Alert Levels framework:
  - **Level 1 (Routine):** Low-importance or expected data — no action needed
  - **Level 2 (Notable):** Medium/High importance with potential for guidance revision or market-moving surprise
  - **Level 3 (Critical):** High importance with black swan potential or major deviation from consensus
- If **Level 2 or higher**, immediately:
  1. Log a Signal in [](https://www.notion.so/5d33fce9e6e240f1ad457f72a8459397?pvs=21) with:
     - **Signal:** Descriptive title referencing the event
     - **Source:** Economic Data
     - **Direction:** Based on fundamental assessment
     - **Confidence:** Based on deviation magnitude and market context
     - **Notes:** Impact analysis, affected watchlist names, and which desks should pay attention
  2. Flag **CHARLES** for PMA opportunity assessment — include the event summary and any Kalshi-relevant angles
- If Level 1, no signal or flag needed — routine monitoring only

## 🧠 When triggered from Trade Ideas

If triggered by a new Trade Ideas entry:

- Only act if the page clearly requests Censori input (for example, Censori is tagged or the content requests fundamentals).
- Add a **Fundamental Analysis** section that includes:
  - Macro & Sector Context
  - Earnings & Financials (recent and upcoming)
  - Catalyst Timeline (dated bullets)
- Keep it concise and decision-useful.
